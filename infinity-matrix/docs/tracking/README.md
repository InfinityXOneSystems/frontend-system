# Tracking System Audit Logs

This directory contains automatically generated audit logs for all system activities.

## Log Structure

Audit logs are organized by:
- **Date**: `YYYY-MM-DD/`
- **Event Type**: `workflow/`, `commit/`, `pr/`, `agent/`
- **Timestamp**: ISO 8601 format in filenames

## Log Types

### Workflow Logs
- **Location**: `workflow/`
- **Content**: Workflow execution details, status, duration
- **Format**: JSON with metadata

### Commit Logs
- **Location**: `commit/`
- **Content**: Commit tracking, file changes, author info
- **Format**: JSON with git metadata

### PR Logs
- **Location**: `pr/`
- **Content**: Pull request lifecycle, reviews, approvals
- **Format**: JSON with PR state history

### Agent Logs
- **Location**: `agent/`
- **Content**: Agent/module deployment and execution logs
- **Format**: JSON with execution traces

## Audit Log Schema

```json
{
  "timestamp": "ISO 8601 datetime",
  "event_type": "workflow|commit|pr|agent",
  "event_id": "unique identifier",
  "actor": "username or system",
  "action": "specific action taken",
  "details": {
    "...": "event-specific details"
  },
  "status": "success|failure|pending",
  "metadata": {
    "workflow_run_id": "if applicable",
    "commit_sha": "if applicable",
    "pr_number": "if applicable"
  }
}
```

## Retention Policy

- **Active Logs**: Last 90 days retained in repository
- **Archived Logs**: Moved to releases after 90 days
- **Critical Logs**: Security and error logs retained indefinitely

## Access

Audit logs are publicly accessible in the repository for full transparency. Sensitive data is never logged.

## Automation

All logs are generated automatically by the `audit-logger.yml` workflow. Manual log entries are not permitted to ensure integrity.
