### Tracking Workflow
**File**: `tracking.yml`  
**Purpose**: Automatically track all repository changes  
**Triggers**:
- Push events to any branch
- Pull request events (open, sync, reopen, close)
- Manual workflow dispatch

**Operations**:
1. Creates tracking logs for commits in `docs/tracking/commit/`
2. Creates tracking logs for PRs in `docs/tracking/pr/`
3. Records workflow executions in `docs/tracking/workflow/`
4. Updates tracking index with latest activity
5. Auto-commits tracking logs to repository

**Outputs**: JSON log files with timestamped activity records

### Project Board Sync Workflow
**File**: `project-board-sync.yml`  
**Purpose**: Synchronize Issues and PRs with GitHub Project Board  
**Triggers**:
- Issue events (open, reopen, close, assign, label)
- Pull request events (open, reopen, close, ready_for_review, draft, review_requested)

**Operations**:
1. Adds new Issues/PRs to Project Board
2. Determines appropriate column based on state
3. Logs synchronization activity
4. Updates project board status

**Column Mapping**:
- **To Do**: New items, reopened items
- **In Progress**: Assigned issues, draft PRs
- **Review**: PRs ready for review
- **Done**: Closed/merged items

## Workflow Monitoring

### Success Criteria
- All workflows complete without errors
- Logs are generated and committed
- No missed events or triggers

### Failure Handling
- Workflows retry automatically on transient failures
- Persistent failures generate alerts in workflow summary
- Manual intervention required for configuration errors

### Performance Metrics
- Average execution time: < 2 minutes
- Success rate target: > 99%
- Log generation completeness: 100%

## Maintenance

### Regular Tasks
- Review workflow logs weekly
- Validate audit log completeness
- Check for failed workflow runs
- Update SOP documentation as workflows evolve

### Troubleshooting

**Issue**: Workflow fails to commit logs  
**Solution**: Check branch protection rules and bot permissions

**Issue**: Project board sync fails  
**Solution**: Verify project board URL and access token

**Issue**: Audit logs missing  
**Solution**: Check workflow triggers and file permissions
